Object Oriented 개념
    현실 세계의 물체를 객체라 생각하였을 때 
    모든 측면을 객체라 표현하지 않고
    중요한 측면 , 주목하고 싶은 측면만을 강조하여 객체로 표현

    추상화 - Class , Object
    캡슐화 - getter / setter
    상속 - extends
    다형성 - Override , Overload
    
  객체와 메시지 - 객체들은 서로 메시지를 주고받음. - 명사 object (객체) 와 동사 event (메시지) 의 구분을 명확히하면 쉬움
  
  Object Oriented 중요 개념 - Object 객체 , Class 클래스 , Inheritance 상속 , 
                              Encapsulation 캡슐화 , Polymorphism 다형성 , Message 메시지
    추상화 기술 - Object -> Class
    상속 - 재사용성
    캡슐화 - 모듈화 , 정보 은닉
    다형성 - 메소드 오버라이딩 & 메소드 오버로딩
Object Oriented Language
  위의 추상화 , 상속 , 캡슐화 , 다형성 등을 더 자세하게 설명함.
  
  추상화 - 객체 모델링 중 필요한 부분만 적용시키는 기술
  
    Object 와 Procedure 들의 공통의 특징들을 골라내는 과정
    
  캡슐화 - 클래스 내부에 여러 Attribute 와 여러 Method 를 함께 묶는 것
  
    클래스 내부의 Attribute 와 Method룰 외부에 노출하지 않고 보호함.
    캡슐화를 통해 정보 은닉을 달성
    소프트웨어를 깔끔하고 복잡하지 않게 만듬
    
  상속 - 클래스 계층을 토대로 클래스 내의 Data 와 Method 를 물려받아 새로운 객체를 생성함
  
    Parent Class / Super Class - 일반적인 범위의 Class
    와 Child Class / Sub Class  - 세부 분류된 Class
    로 분류될 수 있음
    
    중복 코딩의 감소 , 병형성 보장 , 판독성 및 이해성 향상
